MODULE 1 — Panier dans /menu (cart.service)
================================================================================

1.1  Ajout d'un item au panier
     Avec client@restoqr.ca : password123
     - Se connecter et accéder à /menu
     - Cliquer sur un item (ex: « Tartare de Saumon »)
     - La modal s'ouvre avec quantité = 1 par défaut
     - Cliquer sur « Ajouter au panier »
     - Le badge du panier dans le header se met à jour (+1)
     - Le sous-total dans le panneau de droite affiche le bon montant

1.2  Modifier une ligne du panier
     - Ouvrir le panier (icône panier dans le header)
     - Cliquer sur le bouton « Modifier » d'une ligne en attente
     - Une fenêtre de dialogue s'ouvre avec la quantité actuelle et la note
     - Modifier la quantité (ex : de 1 à 3) et/ou la note (ex : « Sans sel »)
     - Cliquer sur « Sauvegarder »
     - La ligne dans le panier est mise à jour (quantité, note, sous-total)
     - Aucun appel API n'est fait : la modification est en mémoire uniquement

1.3  Supprimer une ligne du panier
     - Ouvrir le panier
     - Cliquer sur le bouton « Supprimer » d'une ligne
     - Une fenêtre de confirmation s'ouvre avec le nom de l'article
       et un avertissement : « Cet article sera retiré du panier »
     - Cliquer sur « Supprimer »
     - La ligne disparaît du panier, le badge et le sous-total se mettent à jour
     - Aucun appel API n'est fait : suppression en mémoire uniquement

1.4  Bouton Continuer (Aller à la commande)
     - Avec au moins 1 item dans le panier
     - Cliquer sur « Continuer »
     - Redirection vers /order

1.5  Bouton Continuer désactivé si panier vide
     - Panier vide
     - Le bouton « Continuer » n'est pas visible ou est désactivé

MODULE 2 — Page /form — Sélection du vibe et création de la commande
================================================================================

2.1  Accès à la page /form
     Avec client@restoqr.ca : password123
     - Scanner un QR code ou accéder directement à /form
     - Le formulaire affiche : « Nombre de personnes » et « Type d'ambiance »
     - La liste des vibes actifs (non archivés) est chargée depuis l'API
       GET /api/vibes

2.2  Champ Nombre de personnes — validation
     - Laisser le champ vide → le bouton « Aller au menu » est désactivé
     - Entrer 0 → validation min(1) échoue, bouton désactivé
     - Entrer une valeur valide (ex : 2) → validation passe

2.3  Champ Vibe — obligatoire
     - Le champ vibe est requis (Validators.required)
     - Si aucun vibe n'est sélectionné → le bouton « Aller au menu »
       est désactivé
     - Sélectionner un vibe dans la liste déroulante → bouton s'active
     - Les vibes archivés (deleted_at non null) ne s'affichent pas
       dans la liste

2.4  Prévisualisation de la couleur du vibe
     - Quand un vibe est sélectionné, sa couleur s'affiche visuellement
       (ex : cercle coloré à côté du nom)

2.5  Création de la commande au clic sur « Aller au menu »
     - Remplir un nombre de personnes valide et sélectionner un vibe
     - Cliquer sur « Aller au menu »
     - L'application crée automatiquement une commande via l'API :
       POST /api/orders avec { nb_people, table_id, vibe_id }
     - Redirection automatique vers /menu après la création

2.6  Affichage du vibe dans /order et /kitchen
     - Un badge coloré (ex : « ✦ Festive ») s'affiche à côté du nom
       du serveur dans /order et dans /kitchen

MODULE 4 — Page /order — Affichage et gestion des lignes
================================================================================

4.1  En-tête de la commande
     - Le titre « Votre commande » est visible
     - Le nom du serveur assigné et le badge vibe sont affichés
     - Un bouton « Ajouter + » est visible en haut à droite

4.2  Lignes de commande
     - Chaque item du panier apparaît comme une carte
     - Affiche : nom, quantité × prix unitaire, total, image, note, statut

4.3  Modifier une ligne en attente (panier)
     - Cliquer sur « Modifier » sur une ligne « En attente »
     - Modifier la quantité et/ou la note → mise à jour en mémoire
     - Aucun appel API

4.4  Modifier une ligne envoyée (status = 'sent')
     - Cliquer sur « Modifier » sur une ligne « Envoyée »
     - Modifier, cliquer « Sauvegarder »
     - Appel API : PUT /api/orders/:id/order_lines/:line_id
     - La ligne est mise à jour côté serveur

4.5  Bouton Modifier désactivé sur lignes avancées
     - Les lignes avec statut « En préparation », « Prête » ou « Servie »
       ne sont pas modifiables depuis /order

4.6  Supprimer une ligne en attente (panier)
     - Confirmer → suppression en mémoire uniquement (aucun appel API)

4.7  Supprimer une ligne envoyée (status = 'sent') — hard delete
     - Fenêtre de confirmation avec avertissement de suppression permanente
     - Confirmer → DELETE /api/orders/:id/order_lines/:line_id
     - La ligne est supprimée définitivement

MODULE 7 — Page /history — Historique des commandes passées
================================================================================

7.1  Accès à la page /history
     Avec client@restoqr.ca : password123
     - Accéder à /history
     - Seules les commandes fermées (ended_at non null) sont affichées
     - Chaque commande affiche : table, date, total, lignes, serveur, vibe

7.2  Page vide
     - Si aucune commande fermée → message d'absence d'historique

7.3  Recherche plein texte
     - Champ de recherche disponible (debounce 350ms)
     - Filtre sur : numéro de table, nom d'item, nom du serveur,
       nom du vibe, note, total, date
     - Exemples : « 3 » / « table 3 » → table 3
                  « Tartare » → commandes contenant cet item

7.4  Tri des commandes
     - Sélecteur : « Plus récent » / « Total croissant » / « Total décroissant »
     - Changer le tri relance le chargement

7.5  Filtre par total (sliders min/max)
     - Deux curseurs min et max pour filtrer par montant
     - Le max est calculé dynamiquement selon les commandes du client
     - Saisie directe aussi possible

7.6  Persistance des filtres dans l'URL
     - Les filtres (recherche, tri, min, max) sont synchronisés
       dans les query params → partager l'URL restitue les mêmes filtres

7.7  Évaluation d'un item depuis /history
     - Cliquer sur « Évaluer » sur une ligne → dialogue étoiles + commentaire
     - Appel API : POST /api/reviews
     - L'évaluation apparaît sur la ligne
     - Update via « Modifier », suppression via « Supprimer »

7.8  Évaluation du serveur depuis /history
     - Bouton « Évaluer le serveur » en bas de la carte de commande
     - Même flux que pour un item

MODULE 8 — Page /server — Tableau de bord du serveur
================================================================================

8.1  Accès à la page /server
     Avec server@restoqr.ca : password123
     - Accéder à /server
     - Le tableau de bord affiche :
       « Commandes non assignées » et « Mes commandes »

8.2  Accès refusé
     - Sans être connecté → redirection vers /login
     - Avec un compte client → redirection (serverGuard)

8.3  Section « Commandes non assignées »
     - Commandes ouvertes sans serveur assigné
     - Chaque carte affiche : table, personnes, vibe, lignes avec statuts
     - Bouton « Prendre en charge » disponible

8.4  Assigner une commande
     - Cliquer sur « Prendre en charge »
     - Appel API : POST /api/server/orders/:id/assign
     - La commande passe dans « Mes commandes »

8.5  Servir une ligne (ready → served)
     - Dans « Mes commandes », une ligne « Prête » a un bouton « Servir »
     - Cliquer sur « Servir »
     - Appel API : PATCH /api/server/order_lines/:id/serve
     - La ligne passe au statut « Servie »
     - Quand toutes les lignes sont servies, le client voit
       le bouton « Payer » dans /order

8.6  Modifier une ligne (quantité/note)
     - Cliquer sur « Modifier » sur une ligne non servie
     - Appel API : PATCH /api/server/order_lines/:id
     - La ligne est mise à jour

8.7  Supprimer une ligne — hard delete
     - Cliquer sur « Supprimer » (statut sent ou in_preparation)
     - Fenêtre de confirmation
     - Appel API : DELETE /api/server/order_lines/:id
     - La ligne est supprimée définitivement

8.8  Libérer une commande (non payée)
     - Cliquer sur « Libérer la table »
     - Appel API : POST /api/server/orders/:id/release
     - La commande revient dans « Commandes non assignées »

8.9  Nettoyer une table (commande payée)
     - Une commande payée (ended_at présent) reste visible dans
       « Mes commandes » avec le bouton « Nettoyer la table »
     - Cliquer sur « Nettoyer la table »
     - Appel API : POST /api/server/orders/:id/clean
     - La commande disparaît du tableau de bord

MODULE 10 — Page /admin/vibes — Gestion CRUD des ambiances
================================================================================

10.1 Accès à la page /admin/vibes
     Avec admin@restoqr.ca : password123
     - Accéder à /admin/vibes
     - La liste de toutes les ambiances s'affiche (actives et archivées)
     - Chaque vibe affiche : nom, pastille de couleur, statut

10.2 Accès refusé
     - Sans être connecté → redirection vers /login
     - Avec un compte client ou cuisine → redirection (adminGuard)

10.3 Créer un vibe
     - Cliquer sur « Ajouter une ambiance »
     - Fenêtre de dialogue : Nom (requis, max 50 car., unique),
       Couleur (requis, color picker), Image (optionnel)
     - Cliquer « Créer »
     - Appel API : POST /api/vibes
     - Le nouveau vibe apparaît dans la liste
     - Validation :
       · Nom vide → erreur, formulaire bloqué
       · Couleur manquante → erreur, formulaire bloqué
       · Nom déjà existant → erreur retournée par le serveur

10.4 Modifier un vibe
     - Cliquer sur « Modifier » sur un vibe existant
     - La fenêtre s'ouvre avec les valeurs pré-remplies
     - Modifier le nom, la couleur et/ou l'image
     - Appel API : PATCH /api/vibes/:id
     - Le vibe est mis à jour dans la liste

10.5 Supprimer un vibe non utilisé — hard delete
     - Cliquer sur « Supprimer » sur un vibe sans commande (in_use: false)
     - Fenêtre de confirmation (suppression définitive)
     - Appel API : DELETE /api/vibes/:id
     - Le vibe est supprimé physiquement

10.6 Archiver un vibe utilisé — soft delete
     - Cliquer sur « Supprimer » sur un vibe avec commandes (in_use: true)
     - La fenêtre indique « Archiver » avec l'avertissement :
       « Ce vibe est utilisé par des commandes existantes.
         Il sera archivé et ne sera plus disponible pour de
         nouvelles commandes. »
     - Appel API : DELETE /api/vibes/:id
     - Le vibe reçoit un deleted_at
     - Il reste visible dans la liste avec un badge « Archivé »
     - Il n'apparaît plus dans la liste déroulante de /form

10.7 Restaurer un vibe archivé
     - Sur un vibe archivé, cliquer sur « Restaurer »
     - Fenêtre de confirmation
     - Appel API : PATCH /api/vibes/:id/restore
     - Le deleted_at est remis à null
     - Le vibe est de nouveau actif et réapparaît dans /form

MODULE 11 — Hard delete vs Soft delete
================================================================================

11.1  Lignes de commande — Hard delete
      - Depuis /order : DELETE /api/orders/:order_id/order_lines/:id
        (seulement statut « Envoyée »)
      - Depuis /server : DELETE /api/server/order_lines/:id
        (statuts « Envoyée » ou « En préparation »)
      - Depuis /kitchen : DELETE /api/kitchen/order_lines/:id
        (statuts « Envoyée »)
      - Suppression définitive, non récupérable

11.2  Lignes du panier (en attente) — Suppression en mémoire
      - Aucun appel API (la ligne n'est pas encore en base)
      - Perte des items si rechargement de page avant envoi

11.3  Commandes — Fermeture logique (via ended_at)
      - Jamais supprimées physiquement
      - Fermées en remplissant ended_at
      - Restent accessibles dans /history
      - Endpoints : POST /api/orders/:id/pay
                    POST /api/orders/close_open

11.4  Vibes — Soft delete conditionnel (via deleted_at)
      - Sans commandes → hard delete (supprimé physiquement)
      - Avec commandes → soft delete (deleted_at = now)
      - Restauration possible via PATCH /api/vibes/:id/restore
      - Les vibes archivés n'apparaissent pas dans /form
